Data Preparation

Sample Data: Download sample dataset (xlsx)

Note: this is a snippet—not the full data cleanup process.

Problem

The raw Goals field contains multiple values per client in one comma-separated string. This prevents any meaningful aggregation—e.g. you can’t count how many clients set “Retirement Planning” as a goal or calculate the average number of goals per client.

Objective

  • Normalise the data so each record is atomic (one client + one goal)
  • Support distribution analysis of goals per client
  • Allow easy joins to other tables for cohort or segmentation analysis

Approach

  • Split the Goals column by comma into provisional columns (Goals.1…Goals.N)
  • Unpivot those split columns into a tall format (one row per client-goal pair)
  • Clean each goal value by trimming whitespace for consistency
  • Enrich by joining to a goals reference table to replace IDs with descriptive names

Result

  • Transformed a single multi-value field into a fully normalised, tall table
  • Mapped goals back to the main dataset for accurate reporting


Key M Code Snippet

// Split Goals into separate columns
#"Split Column by Delimiter" = Table.SplitColumn(
    Source,
    "Goals",
    Splitter.SplitTextByDelimiter(",", QuoteStyle.Csv),
    {"Goals.1","Goals.2","Goals.3", /* … Goals.N */ }
),
// Unpivot split columns into rows
#"Unpivoted Columns" = Table.UnpivotOtherColumns(
    #"Split Column by Delimiter",
    {"Client"},
    "Attribute",
    "Value"
),
// Trim whitespace from each goal
#"Trimmed Text" = Table.TransformColumns(
    #"Unpivoted Columns",
    {{"Value", Text.Trim, type text}}
),
// Join to reference table for descriptive names
#"Merged Queries" = Table.NestedJoin(
    #"Trimmed Text",
    {"Value"},
    Goals_Reference,
    {"Goal ID"},
    "GoalsRef",
    JoinKind.LeftOuter
),
#"Expanded GoalsRef" = Table.ExpandTableColumn(
    #"Merged Queries",
    "GoalsRef",
    {"Goal Name"},
    {"Goals"}
)
